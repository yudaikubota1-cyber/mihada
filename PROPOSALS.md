# PROPOSALS — 大きめの改善案
> 実装には時間がかかるため、翌朝以降に優先度を決めてから着手してください。

---

## P1 — 商品画像の実装 🔴 最優先

**現状**：ブランド名のカラープレースホルダー（色のブロック）
**提案**：実際の商品画像URLを `products.js` の各商品に `imageUrl` フィールドで追加

```js
// products.js に追加するフィールド例
{
  id: 'anua-toner',
  imageUrl: 'https://example.com/anua-toner.jpg',
  // ...
}
```

`ProductImage` コンポーネントを `imageUrl` が存在する場合は `<img>` を使うよう改修するだけで対応できます。画像はブランドの公式サイト・楽天商品ページから取得するか、ユーザーがアップロードした素材を使用。

**効果**：体感でサービスのリアリティが10倍になる。

---

## P2 — 診断結果の保存・再表示 🟠 高優先

**現状**：チャットを閉じると診断結果が消える
**提案**：`localStorage` に最新の診断結果を保存し、ホーム画面に「前回の診断を見る」バナーを追加

```js
// 診断完了時に保存
localStorage.setItem('skinr_last_diagnosis', JSON.stringify(diagnosis));
localStorage.setItem('skinr_last_date', new Date().toISOString());
```

ホーム画面に「前回の診断：乾燥肌 · 2日前」→ クリックで結果画面に戻れる

**効果**：リピート率・継続利用につながる。

---

## P3 — お気に入り機能 🟡 中優先

**現状**：気になった商品をどこにも保存できない
**提案**：商品詳細に♡ボタンを追加、`localStorage` で管理
ホーム画面に「お気に入り」フィルタータブを追加

**効果**：ユーザーの滞在時間とエンゲージメントが上がる。

---

## P4 — スキンケアルーティン画面の独立化 🟡 中優先

**現状**：ルーティンが商品詳細の中にある（結果画面から移動してきたもの）
**提案**：「マイルーティン」として独立した画面を作成。診断結果の皮膚タイプ・悩みに基づいて自動生成されるパーソナライズされたルーティンを表示

```
朝のルーティン（4ステップ）
夜のルーティン（5ステップ）
各ステップに商品を紐付け
```

**効果**：ECとしての差別化になる。このルーティンから購入につなげる導線が強い。

---

## P5 — 商品ページから「この悩みの人はこれも見ています」 🟢 低優先

**現状**：商品詳細から他商品への導線がない
**提案**：`concerns` が重なる商品を最大3件、商品詳細の下部に「一緒に使われている商品」として表示

**効果**：客単価アップ（複数商品の同時購入）につながる。

---

## P6 — AIチャットの会話品質向上 🟢 低優先

**現状**：最大3ターンで診断確定
**提案**：システムプロンプトの充実
- 季節・地域（気候）を考慮した成分提案
- 現在使っているスキンケアとの相性チェック
- 「今日から始めるなら1本だけ」プランの提案

**効果**：診断の精度と納得感が上がる。

---

## P7 — SNSシェア機能 🟢 低優先

**現状**：診断結果はアプリ内にしかない
**提案**：結果画面に「診断結果をシェア」ボタン。canvas or OGP画像生成で「私の肌は乾燥肌。おすすめ成分：5Dヒアルロン酸」みたいなカード画像をSNSシェア

**効果**：バイラル拡散。獲得コスト0のマーケティングになる。
